Effets de bord

Considrez l'expressions suivante :

carre 2 + carre 10

Qu'est-ce qui est execut avant : carre 2 ou carre 10 ?

Et comment le vrifier ?

Une fonction qui n'a aucun effet sur son environement est appele pure. Dans ce cas l, l'ordre d'execution n'a pas d'importance, ce qui est un trs grand avantage.

Sinon, on dit qu'elle a des effets de bord.

  • crire sur la sortie standard,
  • Lire sur l'entre standard,
  • crire ou lire un fichier,
  • Renommer un fichier ou un dossier
  • Envoyer un paquet rseau

Le type unit

#show unit ;;

Le type unit est un type qui n'a qu'une valeur possible : (). Il sert au fonction qu'un souhaiteraient ne rien retourner car leur unique but est de faire un effet de bord.

Structures de donnes mutables

Un autre effet de bord possible est la mutation de structures de donnes mutables.

Sauf mention explicite, les structures de donnes fonctionnelles ne sont pas mutables !

Nous allons voir deux structures de donnes mutables : les tableaux et les rfrences.

Tableaux

Les tableaux sont une structure de donne classique en programmation imprative.

let mon_tableau = [| 1; 2; 3 |]
let x = mon_tableau.(1)
let () = mon_tableau.(1) <- 5
let y = mon_tableau.(1)

Quelle valeur x ligne 2 ? Quelle valeur y ? Quelle valeur x ligne 4 ?

val Array.make : int -> 'a -> 'a array
val Array.init : int -> (int -> 'a) -> 'a array

Rfrences

Une rfrence est similaire un tableau qui ne pourrait contenir qu'une seule entre.

let reference = ref 5

let x = !reference

let () = reference := 10

let y = !reference

let reference = ref 5

let reference_2 = reference

let () = reference := 10

let y = !reference_2
